# ST Retrocounter (ESP32)

Прошивка для ретро-счётчика на ESP32: веб-интерфейс в локальной сети, два шаговых мотора 28BYJ-48 (счётчик и диск). **Модуль получения данных из внешнего API или сервиса вы реализуете сами** — в скетче остаётся только точка входа и опциональный HTTP-эндпоинт.

Репозиторий на GitHub: [lucky-strike7/ST-retrocounrter](https://github.com/lucky-strike7/ST-retrocounrter).

## Как зайти в интерфейс

1. В скетче укажите `ssid` и `password` своей Wi‑Fi сети.
2. Загрузите прошивку и при необходимости файлы из папки `data/` (LittleFS / SPIFFS — как настроена схема разделов в Arduino IDE).
3. В браузере откройте:
   - **по mDNS:** [http://ST-retrocounter.local](http://ST-retrocounter.local)  
     (если ОС/network не резолвят `.local`, используйте IP из Serial Monitor после подключения к Wi‑Fi).
   - **по IP:** `http://<IP_ESP32>/` (IP печатается в монитор порта при старте).

## Кратко: что делает интерфейс

- Показывает текущее **целое значение счётчика** (`counterValue`), которое задаёт ваш код через `notifyCounterValue` или через POST `/counterValue`.
- Кнопки **+ / −** и выбор шага (1/10, 1/4, 1, 10) крутят мотор счётчика (и при включённом «диске» — ведомый диск) для ручной подстройки.
- В шапке — включение моторов «счётчик / диск», скорости в RPM, шагов на одну «цифру».
- Внизу — строка **статуса источника данных** (`dataSourceLine`), если вы её задаёте через `setDataSourceStatus`.

## Пины и питание

| Назначение        | GPIO (ESP32)     |
|-------------------|------------------|
| Мотор **диска**   | 5, 18, 19, 21    |
| Мотор **счётчика**| 14, 27, 26, 25   |

- На каждый мотор нужен **драйвер ULN2003** (или аналог) с питанием **5 V**, общая земля **GND** с ESP32. Сигнальные линии — 3.3 V с GPIO (как в типовых схемах с ULN2003).

## Как отдавать данные в прошивку

### 1. Из вашего кода на ESP32 (основной способ)

Вызовите при каждом новом **абсолютном** целочисленном значении счётчика (не «дельту за раз», а полное число, например текущее значение с вашего API):

```cpp
// Первый вызов только запоминает базу, моторы не крутятся.
// Следующие вызовы: прокрутка на (newValue - предыдущее_логическое) × stepsPerDigit шагов.
notifyCounterValue((long)12345);
```

Опционально — короткий текст для подписи внизу экрана:

```cpp
setDataSourceStatus("OK, обновлено 12:00");
```

Если моторы заняты или в очереди команда из веба, `notifyCounterValue` **пропускается** (смотрите сообщение в Serial).

Логическое значение учитывает и **дробные ручные шаги** из веба (1/10, 1/4), чтобы следующий `notifyCounterValue` не «дублировал» уже сделанные повороты.

### 2. По HTTP (для тестов или внешнего сервиса в той же сети)

`POST /counterValue`  
Тело: JSON с целым числом:

```json
{ "value": 12345 }
```

Ответ — текстовый статус (как `/status`). Если устройство занято — вернётся `409`.

### 3. Состояние для своего UI или отладки

`GET /api/state` — JSON, в том числе:

- `counterValue` — текущее отображаемое значение (−1, если база ещё не задана `notifyCounterValue`);
- `dataSourceLine` — строка статуса источника;
- `busy`, `statusLine`, настройки моторов и т.д.

## Локальный превью интерфейса без ESP32

Из корня репозитория:

```bash
python dev_server.py
```

Откройте [http://127.0.0.1:8080/](http://127.0.0.1:8080/) — подставляется mock-ответ `/api/state` (моторы не крутятся).
